##Weekly-I-Learned

스프링에서 컨트롤러 계층은 애플리케이션의 가장 앞단에서 클라이언트의 HTTP 요청을 받고 응답을 돌려주는 역할을 한다는 것을 배웠다. 컨트롤러는 웹 애플리케이션의 진입점으로서, @Controller 어노테이션으로 스프링 빈으로 등록되며, RESTful API를 구축할 때는 @ResponseBody가 결합된 @RestController를 주로 활용한다. 요청을 처리할 적절한 메서드와 URL을 연결하기 위해 @RequestMapping, @GetMapping, @PostMapping 등의 어노테이션을 사용한다. 컨트롤러는 @RequestBody를 통해 요청 본문의 JSON 데이터를 DTO로 변환하거나 @PathVariable을 이용해 URL 경로의 값을 추출한다. 중요한 점은 컨트롤러는 애플리케이션의 핵심 로직인 도메인과 직접 소통하지 않고, 복잡한 로직은 반드시 서비스 계층으로 위임한다는 것이다. 이처럼 컨트롤러는 요청/응답 처리와 라우팅에 집중함으로써 계층을 명확히 분리한다. 서비스 계층으로의 위임은 의존성 주입(DI)을 통해 이루어진다. private final 필드와 @RequiredArgsConstructor를 사용하여 스프링 컨테이너가 생성한 서비스 빈을 컨트롤러에 주입받는 생성자 주입 방식을 배웠다. 이 방식은 컨트롤러와 서비스 간의 결합도를 낮추는 가장 권장되는 DI 방법이라고 한다.
